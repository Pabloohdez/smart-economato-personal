# Glosario

En este glosario hemos definido términos de dominio (en español) y conceptos técnicos que usamos en el proyecto.

## Vocabulario de dominio (rápido)

- **Alérgenos**: sustancias que causan reacciones alérgicas; se controlan por producto y perfil de usuario.
- **Avisos**: alertas (stock bajo, lotes caducados, riesgo económico).
- **Bajas**: retirada de stock por caducidad/rotura/ajuste/merma.
- **Escandallos**: coste de recetas (ingredientes + cantidades + costes).
- **Lote**: una entrada concreta de un producto con cantidad y caducidad.
- **Merma**: pérdida al procesar (bruto vs neto).
- **Rendimiento**: porcentaje útil obtenido tras procesar una materia prima.

## Conceptos técnicos (en sencillo)

- **SPA**: app que navega sin recargar página.
- **API REST**: endpoints del servidor para enviar/recibir datos (JSON).
- **JWT**: token firmado que identifica al usuario.
- **Refresh token rotation**: el token de refresh cambia en cada renovación (más seguro).
- **Transacción**: operaciones que se guardan todas o ninguna.
- **SSE**: canal servidor→cliente para eventos en tiempo real.
- **BroadcastChannel**: sincroniza varias pestañas del navegador.
- **Service Worker**: cache/offline y aceleración de carga.

## Nota

Si en la presentación decimos un tecnicismo, lo traducimos a “palabras normales” usando estas definiciones.

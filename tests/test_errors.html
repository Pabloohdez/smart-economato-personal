Probar Error 503

Cómo Provocar Errores Reales

  • 400 (Datos Inválidos): Envía un POST a /api/login sin username o password
  • 401 (No Autorizado): Intenta login con credenciales incorrectas en /api/login
  • 403 (Prohibido): Accede a /api/auditoria sin ser admin (sin usuario_actual o sin permisos)
  • 404 (No Encontrado): Navega a cualquier URL que no exista, ej: /cualquier-cosa.html
  • 500 (Error Servidor): Si la base de datos (Supabase) no está disponible, /api/productos devolverá error
  • 503 (Mantenimiento): Edita .htaccess y añade ErrorDocument 503 /errors/503.html
Nota Importante: El backend NestJS devuelve siempre JSON con formato { success, error: { message, code } }.
